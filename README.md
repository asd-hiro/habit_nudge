# BiasHack
### 〜行動経済学に基づき「意志力」をハックする学習習慣化アプリ〜

---

## アプリケーション概要
「目標はあるのに、ついスマホを触ってしまう」「二度寝で午前中を無駄にしてしまう」
本アプリは、人間の不合理な行動を「行動経済学」の理論と「IT技術」で制御し、学習習慣を強制的に定着させる自己管理システムです。

## 利用方法
1. **ユーザー登録**: 新規登録時に、あなた自身の名前と、相棒となるキャラクターの名前を決定します。
2. **運命のスロット**: メインロビーでスロットを回し、「今日最初のアクション」を決定します（意志の力を温存するため、1日1回に制限）。
3. **ルーティン登録**: 「Rubyの教材を進める」など、習慣化したいタスクを登録します。
4. **学習記録（計測）**: ルーティン一覧の「学習開始」を押し、終了時に「完了して達成！」を押すことで、実際の学習時間に応じた経験値（EXP）をキャラクターが獲得します。
5. **ペナルティの回避**: 最終学習から24時間以上経過するとキャラクターが「衰退モード」となり、経験値が減少します。この「損失回避」の心理を利用して継続を促します。

## アプリケーションを作成した背景
大学時代に専修した「行動経済学」の理論を、自分自身の課題解決に適応させたいと考えたことがきっかけです。
- **解決したい課題**: 朝の低い判断力が招く二度寝や、将来の利益より今の快楽を優先する「現在志向バイアス」。
- **ターゲット**: 精神論での習慣化に限界を感じている学習者。
- **解決策**: システム側で「行動せざるを得ない状況（ペナルティ）」と「決断を不要にする環境（スロット）」を強制的に提供します。

---

## 実装した機能についての説明

### 1. 損失回避（Loss Aversion）ペナルティ
24時間以上学習記録がない場合に、キャラクターの経験値が減少するロジックを実装。人間が「得る利益よりも失う痛み」を大きく感じる心理特性を逆手に取り、継続率を高めます。

### 2. 意思決定の自動化（スロット機能）
起床直後の決断コスト（ウィルパワーの浪費）をゼロにするため、今日最初のアクションをランダムに指定。DBとセッションを用いた実行フラグ管理により、1日1回制限を厳格に実装しています。

### 3. 実働時間に連動した成長システム
`StudyLog`モデルにより、学習の開始時刻と終了時刻を記録。単なるチェックリストではなく、実際の「学習時間（分）」を計算し経験値に変換することで、達成感を可視化しています。

## 将来の展望（「学習」から「習慣全般」への拡張）
現在の MVP 実装においても、ルーティン登録機能は特定のジャンルに依存しない設計（名称と時間計測の抽象化）を採用しています。そのため、学習に限らず「掃除」「読書」「筋トレ」といった多様なタスクをハックの対象にすることが可能です。

今後はこの汎用性をさらに活かし、以下のアップデートを予定しています。
- **カテゴリー機能の実装**: 学習、健康、家事など、習慣のジャンルごとにアイコンや色分けを行い、管理の利便性を向上。
- **汎用的な習慣化ハックアプリへのリブランディング**: 「学習管理」という枠を超え、あらゆる習慣化をサポートするセルフマネジメントツールとしての価値を確立。

## 工夫したポイント
- **ロジックのべき等性**: `last_penalty_at` 等のカラムを用い、1日に何度もアクセスしてもペナルティが二重に発生しない「べき等」なバッチ処理的ロジックをコントローラーに組み込みました。
- **UXと理論の融合**: 「スロットで決める」「24時間で減る」という行動経済学的フックを、Railsの標準的なCRUD機能と違和感なく統合させました。
- **独自性の追求**: 既存のTODOアプリにはない「行動経済学の社会実装」をテーマに、バックエンドのロジック構築に注力しました。

## データベース設計

### users テーブル
学習者の基本情報および認証情報を管理します。

| Column             | Type   | Options                   |
| :----------------- | :----- | :------------------------ |
| name               | string | null: false               |
| email              | string | null: false, unique: true |
| encrypted_password | string | null: false               |
| character_name     | string | null: false（初期設定用） |

#### Association
- has_one :character
- has_many :routines

---

### characters テーブル
「損失回避」ロジックの核となる、キャラクターの状態を管理します。

| Column          | Type     | Options                        |
| :-------------- | :------- | :----------------------------- |
| name            | string   | null: false                    |
| level           | integer  | null: false, default: 1        |
| exp             | integer  | null: false, default: 0        |
| condition       | integer  | null: false, default: 0        |
| last_penalty_at | datetime | 重複実行防止用（1日1回判定）   |
| user_id         | bigint   | null: false, foreign_key: true |

#### Association
- belongs_to :user

---

### routines テーブル
ユーザーが設定した「習慣化したいタスク」を管理します。

| Column  | Type    | Options                        |
| :------ | :------ | :----------------------------- |
| content | text    | null: false                    |
| status  | integer | null: false, default: 0        |
| user_id | bigint  | null: false, foreign_key: true |

#### Association
- belongs_to :user
- has_many :study_logs

---

### study_logs テーブル
各ルーティンに紐づく、具体的な学習記録を管理します。

| Column           | Type     | Options                        |
| :--------------- | :------- | :----------------------------- |
| started_at       | datetime |                                |
| ended_at         | datetime |                                |
| duration_minutes | integer  |                                |
| focus_score      | integer  |                                |
| comment          | text     |                                |
| routine_id       | bigint   | null: false, foreign_key: true |

#### Association
- belongs_to :routine
