## アプリケーション概要
「目標はあるのに、ついスマホを触ってしまう」「朝、何から手をつけるべきか迷っているうちに時間が過ぎる」
本アプリは、人間の不合理な行動を「行動経済学」の理論（損失回避・ナッジ）で制御し、タスク実行を強制的に定着させるセルフマネジメントシステムです。

## 利用方法
1. **ユーザー登録**: あなた自身の名前と、相棒となるキャラクターの名前を決定します。
2. **運命のスロット**: メインロビーでスロットを回し、「今日最初のアクション」を決定します。意志の力を温存するため、システムが決断を代行します。
3. **タスク登録**: 習慣化したいアクションを「タスク」として登録します。
4. **アクション開始（計測）**: タスクの「開始」ボタンを押し、終了時に「達成！」を押すことで、活動時間に応じた経験値（EXP）をキャラクターが獲得します。
5. **ペナルティの回避**: 最終活動から24時間以上経過すると、キャラクターが「衰退モード」となり経験値が減少します。この「損失回避」の心理を利用して継続を促します。

## アプリケーションを作成した背景
大学時代に専修した「行動経済学」の理論を、自分自身の課題解決に適応させたいと考えたことがきっかけです。
- **解決したい課題**: 朝の低い判断力が招く二度寝や、将来の利益より今の快楽を優先する「現在志向バイアス」。
- **ターゲット**: 精神論での習慣化に限界を感じている学習者。
- **解決策**: システム側で「行動せざるを得ない状況（ペナルティ）」と「決断を不要にする環境（スロット）」を強制的に提供します。

## 実装した機能についての説明

### 1. 損失回避（Loss Aversion）ペナルティ
24時間以上アクションがない場合に、キャラクターの経験値が減少するロジックを実装。人間が「得る利益よりも失う痛み」を大きく感じる心理特性を逆手に取り、継続率を高めます。

### 2. 意思決定の自動化（スロット機能）
起床直後の決断コスト（ウィルパワーの浪費）をゼロにするため、今日最初のアクションをランダムに指定。**スロット結果はデータベースに永続化**されるため、ログアウトしてもその日のミッションは保持されます。

### 3. 実働時間に連動した成長システム
`StudyLog`モデルにより、開始時刻と終了時刻を記録。単なるチェックリストではなく、実際の「活動時間（分）」を計算し経験値に変換することで、達成感を可視化しています。

### 4. 絵文字対応のデータ基盤（MySQL utf8mb4）
ユーザーが直感的に操作できるよう、タスク名やスロット結果に絵文字を使用可能にしました。DB層で `utf8mb4` エンコーディングを採用し、現代的なUXを支える堅牢なデータ保存を実現しています。

## 工夫したポイント
- **ロジックのべき等性**: `last_penalty_at` 等のカラムを用い、1日に何度もアクセスしてもペナルティが二重に発生しない設計にしました。
- **UXの向上**: ログアウトしても「今日のスロット結果」が維持されるよう、セッション管理からDB保存へと移行しました。
- **技術的課題の解決**: MySQLにおける絵文字エラーを、DB構成の根本的な見直し（encoding設定）により解決しました。
